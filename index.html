This project uses a model based on the premise developed by Penna, where bit strings represent individuals' aging. The work is an update of the model by Keesen et al, introducing the simulation of fragmented areas, mimicking real habitat divisions.

Developed in C++ using object-oriented programming, the focus is on creating an accurate representation of virtual individuals. These individuals age, reproduce, and die as they interact with the fragmented environment, exploring the ecological challenges of survival and migration between different areas.

The combination of simulated aging with habitat fragmentation makes this model a powerful tool for understanding the effects of genetics and the environment on population dynamics.

How to use the tool?

Simulation Parameters Configuration

In this section, you can adjust the key parameters that control the population simulation. These parameters define essential aspects of population dynamics, such as the behavior of aging, reproduction, and individual interactions with the environment. By adjusting these values, you can explore different scenarios and observe how genetic, ecological, and spatial factors influence the evolution and survival of a population over time.

The flexibility offered by this configuration allows you to simulate anything from small populations in stable environments to large, fragmented ecosystems with complex challenges of migration and limited resources. The parameters set here are crucial to customizing the model according to the needs of the study or experiment you wish to conduct.

Once you’ve adjusted the desired parameters, simply run the model and observe the results based on the conditions you’ve configured.

Time Configuration

In this section, you can adjust the time parameters that control the duration and time unit used (days, months, years) of the population simulation. Accurate time configuration is essential to ensure that the model simulates population events precisely over different temporal scales. By adjusting these values, you can explore how population dynamics develop over short or long periods, providing detailed insights into the population's behavior over time.

Sampling Configuration

In this section, you can adjust both the number of samples and the number of datasets that the model will generate during the simulation. The number of samples determines how many times the model will run for each dataset, while the number of datasets defines the number of sample sets generated, allowing for the calculation of the standard deviation for each set.

For example, by setting 50 samples and 10 datasets, the model will run 50 times for each dataset, generating 10 sample sets. This setup facilitates statistical analysis of the results, providing a more accurate view of data variability.

Adjust these parameters according to the desired level of detail, keeping in mind that increasing the number of samples and datasets extend processing time.

Environment Configuration

In this section, you can define the characteristics of the environment that will influence the model simulation. The parameters you set here are essential for determining the habitat structure and population dynamics.

Fragments and Area:

You can adjust the number of fragments that make up the environment, as well as specify the total area available for the simulation. Configuring the area and the number of fragments will have a direct impact on how populations interact and distribute.

Perimeter and Shape:

It is possible to define the perimeter and shape of each fragment. These parameters are crucial, as areas with different shapes and boundaries can influence species accessibility and survival.

Initial Population Density:

The initial population density is another vital aspect, as it determines how many individuals will start the simulation. This value can be adjusted based on the ecological characteristics of the environment.

Fragmentation:

You will also have the option to decide whether the area will be fragmented. If you choose to fragment, you can specify how many parts the area will be divided into and at which time step this fragmentation will occur. This fragmentation dynamic is fundamental for simulating realistic ecological scenarios, where habitats may be altered by natural or anthropogenic processes.

Connectivity:

allows for the migration of individuals between different areas. You can specify the degree of connectivity, influencing how easily individuals can move and disperse throughout the habitat. This factor is essential for understanding population dynamics, gene flow, and the overall resilience of species in fragmented landscapes.

Configuring these parameters will allow for a more accurate and representative simulation of environmental conditions, enabling a detailed analysis of population interactions, the effects of fragmentation, and the role of connectivity in migration processes.

Species Configuration

This section defines the characteristics of the species in the model. The parameters provided below help simulate species-specific behaviors and life cycles.

The species section includes parameters such as Name and Longevity, determining the species' lifespan in the simulation. In the Distribution section, you can set the Density, defining the number of individuals per square kilometer. Additionally, you can configure whether the species exhibits gregarious behavior, organizing into groups, and set the maximum group size.

For Reproduction, parameters include the Maturity Age, when individuals become reproductively active, and the Reproductive Success, which affects the likelihood of successful reproduction. You can also define whether the species reproduces sexually or asexually, as well as the average number of offspring per reproductive event.

The Social Hierarchy section allows you to specify if the species has an alpha individual, the duration of the alpha's reign, and the alpha's reproductive success rate. In the Parental Behavior section, you can indicate whether the species provides parental care to their offspring and the duration of this care.

Dynamics configuration

On this section, you can adjust various parameters related to population dynamics. The Verhulst Model is incorporated to control population growth by applying a bias to soften the logistic growth effect. The Movement Dynamic allows you to configure how populations move and spread, defining the age at which individuals disperse, the rate of dispersion, and which gender is responsible for dispersing.

The evolutionary aspect is modeled using the Penna Model, where you can simulate genetic evolution by adjusting parameters such as the mutation rate limit, the size of the genetic string, the number of dominant positions, and penalties for consanguineous reproduction.

Simulation

In this section, you can run the population simulation using the pre-configured parameters. To begin, ensure that the desired parameters have been set in the Parameters section. These parameters determine the behavior and initial conditions of the simulation.

Once the parameters are selected, you can start the simulation, which will calculate the population dynamics based on the defined conditions. The simulation progress will be displayed on the screen, and the results can later be visualized and analyzed in the graphs section.

Assign preconfigured parameters

Graph Generation

In this section, you can visualize the results of the population simulation through detailed graphs. To begin, select the folder where the model’s output files are located. From this data, graphs will be generated to represent population dynamics over time, such as growth, aging, mortality rates, migration, and other key factors.

Once the folder is selected, the graphs will be automatically created based on the provided information, allowing for a visual analysis of the simulated scenarios. This makes it easier to understand emerging patterns and the behavior of the simulated population under different conditions.
